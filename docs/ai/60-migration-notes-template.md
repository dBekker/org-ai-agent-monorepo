# 60. Migration note template

Use this template when a task changes a contract, schema, folder layout, or deployment flow.

## Summary
- What changed?
- Why was it changed?
- Which consumers are affected?

## Risk level
- low / medium / high

## Required follow-up
- config updates
- env updates
- migration commands
- manual QA
- data backfill or content model sync

## Compatibility
- backward compatible / partially compatible / breaking

## Rollout note
- how to deploy safely
- what to monitor right after rollout
