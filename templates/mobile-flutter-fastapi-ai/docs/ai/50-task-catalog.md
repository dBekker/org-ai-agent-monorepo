# 50. Standard task catalog

This file defines the tasks that the team expects coding agents to handle well.

## Web stack

1. Import a page from Figma into the existing architecture.
2. Add a reusable component to the shared UI library.
3. Fix a responsive regression on target breakpoints.
4. Add a CMS-driven section and wire it into the frontend.
5. Add a new Strapi content model or component.
6. Add a Prisma model and migration for app-domain data.
7. Build a form with validation and request states.
8. Refactor a page or module without changing visible behavior.
9. Update Docker and env wiring for a new feature.
10. Leave migration notes and updated docs.

## Mobile / backend stack

1. Import a screen from Figma into a feature module.
2. Fix an adaptive layout issue on multiple device classes.
3. Add a new Flutter feature using data / domain / presentation separation.
4. Add a FastAPI endpoint with schemas, service, and tests.
5. Add a new gateway flow that coordinates other services.
6. Add a new recipes-domain capability.
7. Add a new assistant action end-to-end.
8. Stabilize a websocket or transport contract.
9. Refactor a legacy module toward the target structure.
10. Update docs, risks, and migration notes.
