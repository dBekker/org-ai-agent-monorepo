# Internal Mobile Flutter Overlay Summary

This overlay strengthens the foundation repo for real brownfield Flutter/mobile adoption without polluting the greenfield template.

## Included
- brownfield-aware Flutter architecture guidance
- code-style rules for imports, routing, DI, failures, and code generation
- testing playbooks for unit, repository, BLoC, widget, golden, and integration coverage
- CI and release/deploy checklists

## Main stance
- keep greenfield and brownfield guidance separate
- choose a structure mode before adding new features
- normalize only what is touched
- make generated-files policy explicit
- keep donor lessons generalized and reusable
